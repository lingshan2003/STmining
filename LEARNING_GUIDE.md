# 边写边学指南

## 第一课：时间序列如何变成监督学习

原始速度矩阵是 `V[T,N]`：`T` 个时间点，`N` 个传感器。一个样本选择预测起点 `t`：

- `x = V[t-12:t]`，代码中形状为 `[12,N,1]`；
- `y = V[t:t+12]`，形状为 `[12,N]`；
- `mask` 与 `y` 同形状，表示哪些真实值可以计入损失。

先阅读 `TrafficDataset.__getitem__`。 练习：在纸上取 `T=10, input=3, output=2`，列出所有合法预测起点。

## 第二课：为什么切分容易泄漏

随机打散时间窗口会让未来交通模式进入训练集。本项目先按绝对时间确定 train/validation/test 边界，再按首个目标时间分配样本。验证集可以读取边界前的历史作为输入，但其预测目标不会出现在训练集。

缩放器只查看训练期且非缺失的值。练习：运行 `tests/test_data.py`，再尝试让异常值处的 `mask=True`，观察均值变化。

## 第三课：从共享 MLP 到共享 LSTM

MLP 把每个节点过去 12 个数直接映射为未来 12 个数。LSTM 按时间逐步更新隐藏状态。两者都把节点折叠进 batch，并让所有节点共享参数，因此模型参数量不随节点数增长，也不使用空间关系。

阅读 `SharedLSTM.forward`，逐行写出以下形状变化：

```text
[B,T,N,1] → [B,N,T,1] → [B*N,T,1] → [B*N,H] → [B,T_out,N]
```

## 第四课：图卷积是什么

归一化邻接矩阵 `A[N,N]` 描述传感器连接。矩阵乘法 `AX` 让每个节点获得邻居特征的加权平均；线性层再混合通道。权重形状只含特征通道，不含 `N`，因此同一 checkpoint 可以用于另一个节点数不同的城市。

阅读 `GraphConv.forward` 中的 `einsum`。练习：构造三个节点的链状图，令只有第一个节点特征为 1，检查一次消息传递后的结果。

## 第五课：STGCN 如何组合时间与空间

一个 ST block 执行：门控时间卷积 → 图卷积 → 门控时间卷积 → 归一化。两个 block 后，预测头同时汇总 12 个输入时刻并输出 12 个未来时刻。残差连接帮助梯度传播。

先用 5 个节点的随机数据运行 `tests/test_models.py`，再增加到 8 个节点；参数形状应完全不变。

## 第六课：怎样做可信的迁移实验

- `target-scratch` 检查少量目标数据自身能学到多少；
- `source-zero-shot` 测量未经适配的分布差异；
- `full-finetune` 检验源模型是否提供更好的初始化；
- `temporal-frozen` 检验源城市时间特征是否可复用；
- `target-full` 只是数据充足上界。

对每个设置只改变一个因素，并保持测试集、少样本预算、早停和随机种子一致。迁移增益定义为 `MAE_scratch - MAE_transfer`；负值就是负迁移，不应删除。

## 推荐学习节奏

每完成一部分都遵循：读概念 → 打印一个 batch 的形状 → 跑单测 → 两轮 smoke training → 看曲线 → 修改一个小因素并解释结果。不要一开始运行全部实验矩阵；先证明一条链路可信。
